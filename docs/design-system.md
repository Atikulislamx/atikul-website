# Design System

## Project

ATIKUL WEBSITE

Official Website of **Atikul Islam Rabbi**

---

# Purpose

This document defines the visual language, layout principles, user experience guidelines, and reusable interface components used throughout the website.

Every page should maintain a consistent design system to ensure professionalism, usability, accessibility, and long-term maintainability.

---

# Design Principles

The website should always feel:

* Professional
* Modern
* Minimal
* Trustworthy
* Clean
* Fast
* Accessible
* Consistent

Avoid unnecessary decorations or overly complex layouts.

---

# Design Philosophy

The interface should prioritize:

* Readability
* Simplicity
* Clear navigation
* Visual hierarchy
* Consistency
* Performance

The design should help visitors find information quickly without distractions.

---

# Grid System

Use a responsive layout.

Recommended content width:

* Maximum Width: 1200px
* Content Width: 1100px
* Wide Sections: 1400px (optional)

Center all primary content.

---

# Spacing System

Maintain consistent spacing.

Recommended spacing scale:

* 4px
* 8px
* 12px
* 16px
* 24px
* 32px
* 48px
* 64px
* 96px

Avoid inconsistent margins and padding.

---

# Border Radius

Use subtle rounded corners.

Suggested values:

* Small: 6px
* Medium: 10px
* Large: 16px

Avoid excessive rounding.

---

# Shadows

Use soft shadows only where needed.

Cards:

Light shadow.

Buttons:

Very subtle shadow.

Avoid heavy or unrealistic shadows.

---

## Accent

---

## Status Colors

# Typography

Use clean system fonts.

Preferred stack:

Arial, Helvetica, sans-serif

Prioritize readability over decoration.

---

# Font Scale

Hero Title

48–64px

Page Titles

36–42px

Section Titles

28–32px

Subtitles

20–24px

Body Text

16–18px

Small Text

14px

Caption

12px

Maintain a clear visual hierarchy.

---

# Line Height

Headings

1.2

Paragraphs

1.6

Lists

1.6

Improve reading comfort.

---

# Buttons

Primary Button

* Solid accent color
* White text
* Rounded corners
* Hover effect
* Smooth transition

Secondary Button

* Transparent background
* Accent border
* Accent text

Text Button

* Underlined on hover
* Minimal styling

Buttons should always have visible hover and keyboard focus states.

---

# Cards

Cards should be reusable.

Examples:

Service Card

Blog Card

Portfolio Card

Review Card

Feature Card

Cards should include:

* Consistent padding
* Rounded corners
* Soft shadow
* Hover effect (subtle)

---

# Navigation

Desktop Navigation

Logo

Primary menu

CTA button

Mobile Navigation

Hamburger menu

Slide-down or slide-out navigation

Easy touch targets

Highlight the current page.

---

# Hero Section

The homepage hero should contain:

Professional portrait

Headline

Short introduction

Call-to-action buttons

Optional statistics

Avoid oversized decorative graphics.

---

# Sections

Every section should include:

Title

Optional subtitle

Content

Appropriate spacing

CTA if applicable

Maintain consistent spacing between sections.

---

# Forms

Contact forms should include:

Labels

Placeholder text

Validation

Error messages

Success messages

Accessible inputs

Required fields should be clearly indicated.

---

# Icons

Use one consistent icon style.

Icons should:

Support content

Remain minimal

Avoid visual clutter

---

# Images

Use:

High-quality images

Compressed files

Consistent aspect ratios

Proper alt text

Lazy loading where appropriate

Avoid stretched images.

---

# Blog Layout

Each article should include:

Title

Featured image

Author

Publish date

Reading time

Category

Table of contents (optional)

Content

Related articles

Share buttons

Call to action

---

# Portfolio Layout

Each project card should display:

Project title

Short description

Services provided

Result summary

Optional image

Call-to-action

---

# Review Cards

Each review should include:

Client name

Role or company (if public)

Rating

Review text

Date (optional)

Maintain a clean and consistent appearance.

---

# Footer

Navigation links

Quick links

Contact information

Social media

Privacy Policy

Terms of Service

---

# Animations

Animations should be subtle.

Recommended:

Fade

Slide

Scale

Smooth hover transitions

Avoid distracting effects.

---

# Accessibility

Support:

Keyboard navigation

Visible focus indicators

High color contrast

Alt text

Proper labels

Semantic HTML

Logical heading order

---

# Performance

Optimize:

Images

CSS

JavaScript

Fonts

Avoid unnecessary assets.

Prioritize fast loading.

---

# Component Consistency

Reusable components should maintain consistent:

Spacing

Typography

Colors

Buttons

Cards

Forms

Icons

Navigation

Never create duplicate visual styles for the same component type.

---

# User Experience

The visitor should always know:

Where they are.

What the website offers.

What action to take next.

How to contact the owner.

Navigation should remain intuitive and predictable.

---

# Design Goal

The overall experience should resemble a professional personal brand website rather than a generic portfolio.

Visitors should immediately recognize expertise, trustworthiness, and professionalism while enjoying a clean, modern, and fast user experience.
